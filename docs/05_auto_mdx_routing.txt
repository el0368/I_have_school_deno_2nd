================================================================================
AUTO-MDX ROUTING MECHANISM
================================================================================

This project uses a custom routing system to make .mdx files feel like native
routes. You can drop a markdown file into the routes folder and view it
instantly in your browser.

--------------------------------------------------------------------------------
1. HOW THE ROUTING FLOW WORKS
--------------------------------------------------------------------------------

The system follows a specific order when you request a URL:

1. STANDARD ROUTES:
   The app first checks for regular .tsx files (like index.tsx).
   
2. MDX FALLBACK:
   If no standard page is found, the server checks the "MDX Registry".
   
3. DYNAMIC RENDERING:
   If the path is in the registry, the server loads the .mdx file, wraps it in
   a standard layout, and displays it.

--------------------------------------------------------------------------------
2. WHY WE USE MAIN.TS (NOT MAIN.TSX)
--------------------------------------------------------------------------------

Vite requires the entry point of the application to be "main.ts". 
If we rename it to .tsx to use JSX tags, the build tool may fail to start
the development server.

To stay compatible, we use the "h()" function in main.ts instead of <tags>.
Example: h("div", { class: "py-8" }, h(MDXContent, null))

--------------------------------------------------------------------------------
3. KATEX MATH SUPPORT
--------------------------------------------------------------------------------

You can write mathematical equations directly in your MDX files.

* INLINE MATH: 
  Use single dollar signs. Example: $E = mc^2$
  
* BLOCK MATH: 
  Use double dollar signs on their own lines.
  Example:
  $$
  a^2 + b^2 = c^2
  $$

The required CSS for these symbols is automatically loaded by the global
application layout in "routes/_app.tsx".

--------------------------------------------------------------------------------
4. HOW TO ADD NEW MDX PAGES
--------------------------------------------------------------------------------

1. Create your file: routes/my-page.mdx
2. Register it: Add the path to "mdx-routes.ts"
3. View it: Visit http://localhost:5173/my-page

No other configuration or manual wrapper files are required.
